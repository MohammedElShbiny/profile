## Bug Fixes in api/lib/db.js
1. **`getClient()` null bug** — returns null when client not initialized (line 6-7). Fix: create client when null.
2. **`parseBody()` fails** — does `JSON.parse(req.body)` but Vercel may not populate `req.body`. Fix: use `req.text()` then parse.
3. **`checkAuth()` uses Node Headers** — uses `req.headers.authorization` (Node). Fix: use `req.headers.get('authorization')` (Web Standard).

## Deployment Configs

### package.json
```json
{
  "name": "core-systems-portfolio",
  "version": "4.0.0",
  "description": "Multi-platform serverless portfolio with Turso",
  "private": true,
  "scripts": {
    "dev": "vercel dev",
    "deploy:vercel": "vercel --prod",
    "deploy:netlify": "netlify deploy --prod",
    "db:setup": "node scripts/seed.js"
  },
  "dependencies": {
    "@libsql/client": "^0.14.0"
  }
}
```

### vercel.json
```json
{
  "version": 2,
  "builds": [
    { "src": "api/**/*.js", "use": "@vercel/node" },
    { "src": "public/**", "use": "@vercel/static" }
  ],
  "routes": [
    { "src": "/api/(.*)", "dest": "/api/$1" },
    { "src": "/admin", "dest": "/public/views/admin.html" },
    { "src": "/chat", "dest": "/public/views/chat.html" },
    { "src": "/(.*)", "dest": "/public/$1" }
  ]
}
```

### netlify.toml
```toml
[build]
  publish = "public"
  functions = "netlify/functions"
[[redirects]]
  from = "/api/*"
  to = "/.netlify/functions/api/:splat"
  status = 200
```

### netlify/functions/api.js
Single adapter that converts Netlify event format → standard Request → routes to api/ handler → converts Response back.
