## How Routing Works in Merged Files

Each merged handler parses `req.url` to determine the sub-route:

```js
// api/projects.js
const { getParam } = require('./lib/db');

module.exports = async function handler(req) {
  if (req.method === 'OPTIONS') return corsPreflight();
  const url = new URL(req.url, 'http://localhost');
  const path = url.pathname;

  // GET /api/projects
  if (req.method === 'GET' && path === '/api/projects') { ... }

  // PUT /api/projects/123
  if (req.method === 'PUT' && path.match(/^\/api\/projects\/\d+$/)) {
    const id = path.split('/').pop();
    ...
  }

  // DELETE /api/projects/123
  if (req.method === 'DELETE' && path.match(/^\/api\/projects\/\d+$/)) { ... }
};
```

Same pattern for `api/admin.js` (dispatches to stats/inquiries/messages sub-routes) and `api/chat.js` (dispatches to token/unread sub-routes).
