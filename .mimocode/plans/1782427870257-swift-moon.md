# Plan: Multi-Platform Serverless App with Turso + Local SQLite

## Goal
Keep the app serverless on Vercel (production) with Turso, add an Express adapter for local dev with SQLite, and create a DEPLOY.md guide explaining database setup and deployment across platforms.

## Architecture

```
Production:  Vercel Serverless (api/*.js) → Turso (cloud DB)
Local Dev:   Express Server (server.js)   → SQLite (data/portfolio.db)
```

The `lib/db.js` module auto-detects the environment:
- If `TURSO_DATABASE_URL` is set → use Turso (`@libsql/client`)
- Otherwise → use local SQLite (`sql.js`)

## Files to Create

### 1. `lib/db.js` (replace existing) — Unified database module
- Exports `getClient()` that returns a db object with consistent interface
- Auto-detects Turso vs SQLite based on env vars
- Wraps both backends with `queryAll()`, `queryOne()`, `runSql()` helpers
- Keeps `initDb()`, `seedIfEmpty()`, `checkAuth()` functions
- Keeps `jsonRes()`, `errRes()` helpers for Vercel handlers

### 2. `server.js` — Express adapter for local dev
- Only used for `npm run dev` (local development)
- Serves `public/` as static files
- Mounts API routes under `/api`
- Routes mirror the Vercel handler structure (reuses same lib/db.js)
- Handles CORS, initializes DB on startup

### 3. `routes/` directory — Express route files (6 files)
Each file reuses the same DB helpers from `lib/db.js`:

| File | Endpoints |
|---|---|
| `routes/projects.js` | GET `/projects`, PUT `/projects/:id`, DELETE `/projects/:id` |
| `routes/services.js` | GET `/services`, PUT `/services/:id`, DELETE `/services/:id` |
| `routes/contact.js` | POST `/contact` |
| `routes/lookup.js` | GET `/lookup` |
| `routes/chat.js` | GET/POST `/chat/:token`, GET `/chat/:token/unread` |
| `routes/admin.js` | GET `/admin/stats`, GET `/admin/inquiries`, GET/POST `/admin/messages`, GET/POST `/admin/messages/:inquiryId` |

### 4. `DEPLOY.md` — Deployment guide
